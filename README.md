# PII Firewall

Локальный файрвол для персональных данных: обнаруживает и обратимо маскирует
ПДн/PII **прямо в браузере**, до того как промпт уйдёт в AI-чат
(ChatGPT, Claude, Gemini, Perplexity, Poe), и восстанавливает оригиналы в
ответе. Без сервера, без аккаунта, **без единого сетевого разрешения** —
проверьте `manifest.json`: единственное разрешение `storage` (локальное
хранилище браузера), никакого `host_permissions` к сети.

> **Статус: M1 (движок), M2 (vault), M3 (content UI), M4 (guard),
> M5 (popup/options/i18n), M6 (onboarding) готовы.**
> Остаётся E2E-прогон по всем сайтам и M8 (скриншоты, видео, submission).

## Движок (`src/engine/`)

Чистый TypeScript, ноль runtime-зависимостей, без DOM. Конвейер:

```
текст → NFKC-нормализация + снятие комбинирующих знаков (с картой смещений)
      → фолдинг гомоглифов кириллица→латиница (1:1, для Latin-паттернов)
      → кандидаты (regex) → валидация (контрольные суммы / формат / контекст)
      → remap спанов на исходный текст
      → разрешение перекрытий: длиннейший спан;
        при равенстве checksum > secret > structured > person
```

| Тип | Валидация | Уверенность |
|---|---|---|
| EMAIL | формат (+ фолдинг гомоглифов) | high |
| PHONE | RU `+7/8` с пробелами/дефисами/скобками, intl `+…`; границы «не цифра» | high; голые 11 цифр — medium |
| CARD | 13–19 цифр, Luhn | группировка/контекст — high; голые 16 — medium; прочие голые — low |
| IBAN | mod-97 (+ фолдинг) | high |
| RU_SNILS | mod-101 + резервный диапазон; голые 11 цифр требуют контекста «СНИЛС» | high |
| RU_INN | весовые проверки 10/12 цифр | контекст «ИНН» — high; голые 12 — medium; голые 10 — low |
| RU_OGRN | mod-11/13 (13/15 цифр) | контекст «ОГРН» — high; иначе low |
| RU_PASSPORT | 4+6 цифр + контекст (паспорт/серия/выдан) | medium |
| RU_OMS | 16 цифр, Luhn | контекст «полис/ОМС» — high; иначе medium |
| US_SSN | формат + невалидные диапазоны (000/666/9xx, 00, 0000) | high |
| DATE_OF_BIRTH | **opt-in, выключен по умолчанию** — слишком шумный | контекст рождения — high; иначе medium |
| IP_ADDRESS | IPv4 | medium |
| SECRET_* | JWT, AWS-ключ, key=value, энтропия (hex-порог отдельный) | high/medium |
| PERSON | ФИО-эвристика: словарь ~180 имён со стемматчингом падежей, морфология отчеств/фамилий (включая косвенные формы), «Фамилия И.О.», EN title+имя | high/medium/low |

## Обратимое маскирование (`src/vault/`)

- Плейсхолдеры с полным именем типа: `[PERSON_1]`, `[RU_SNILS_1]`,
  `[SECRET_KV_1]` — квадратные скобки переживают markdown/HTML-санитайзеры
  в ответах AI.
- Одинаковое значение → один плейсхолдер в рамках сессии (стабильная
  кореференция); счётчики продолжаются между документами.
- Vault в `chrome.storage.session` (автоочистка при закрытии браузера),
  ключ — таб + путь чата: restore с чужой сессии **отказывает** и возвращает
  список неизвестных плейсхолдеров. Для смены URL после первого сообщения
  (`chatgpt.com/` → `/c/<id>`) есть `migrateSession`.
- Хранятся только пары значение↔плейсхолдер — исходный текст промпта
  не сохраняется нигде и никогда.
- Плейсхолдеро-подобные строки, уже присутствующие в тексте, не
  переиспользуются (защита от порчи restore при вставке ранее
  замаскированного текста).
- «Забыть»: по сессии и целиком (`clearAll` трогает только `vault_*`-ключи).
- Опциональная персистентность: снапшот vault шифруется **AES-GCM-256**
  с ключом из **PBKDF2-SHA256 (600k итераций)** и лежит в
  `chrome.storage.local`; пароль не хранится, неверный пароль ломает
  GCM-аутентификацию. Без chrome.storage маскирование **громко падает**
  (`VaultStorageError`), а не теряет данные молча.
- Приёмка M2: property-тест round-trip — 120 случайных синтетических
  документов через реальный движок, побайтовое восстановление.

## Content script (`src/content/`)

- Плавающий виджет (closed Shadow DOM, тёмная тема): «Замаскировать» /
  «Восстановить», чипы по типам сущностей, статус.
- Сайты: ChatGPT, Claude, Gemini, Perplexity, Poe — селекторы в одном файле
  `sites.ts`; редактор ищется по селектору сайта (только видимые элементы)
  с геометрическим fallback.
- Запись в редакторы, переживающая реальные сайты: нативный сеттер value +
  `InputEvent` для React-textarea; `selectAll + insertText` для
  ProseMirror/Lexical/Quill.
- После маскирования добавляется инструкция модели «сохраняй метки»
  (идемпотентно); restore снимает её до обращения к vault.
- «Восстановить» с выделением — расшифровка фрагмента ответа в буфер обмена.
- Горячие клавиши через manifest `commands` (переназначаемые):
  Alt+Shift+M — маскировать, Alt+Shift+U — восстановить.
- Смена URL после первого сообщения (`/` → `/c/<id>`) отслеживается,
  mapping мигрирует (`Vault.migrateSession`); переключение между чатами
  сессии не смешивает.
- Service worker открывает `chrome.storage.session` для content scripts
  (`setAccessLevel`) и выдаёт tabId для ключей vault.

Ручная приёмка — [docs/e2e-checklist.md](docs/e2e-checklist.md),
пошагово — [docs/manual-testing.md](docs/manual-testing.md).

## Guard-режим (`src/content/guard.ts`)

- Перехват отправки (Enter без Shift в capture-фазе + клик по кнопке
  отправки) → скан → модал «Замаскировать и отправить / Отправить как есть /
  Отмена» с подсветкой найденных ПДн (весь пользовательский текст
  экранируется — без XSS).
- Глобальный тумблер (настройки) + переключатель на каждый сайт (popup).
- Счётчик «предотвращено утечек» инкрементится при «Замаскировать и
  отправить», виден в popup.

## Popup / настройки / i18n (`src/popup/`, `src/options/`, `_locales/`)

- **Popup:** сайт, счётчик утечек, guard-тумблер для текущего сайта, «Забыть
  сессию / Забыть всё», «Демо-режим» (открывает ChatGPT + синтетический
  пример в буфер), ссылки на настройки и onboarding.
- **Options:** все 14 типов детекции (DATE_OF_BIRTH — opt-in), guard и
  инструкция, язык (авто/ru/en), опциональное шифрованное хранилище
  (AES-GCM, пароль). Настройки читаются content-скриптом **на лету**
  (через `chrome.storage.onChanged`, без перезагрузки).
- **i18n:** RU (основная) + EN через `_locales` и `default_locale`; имя и
  описание в манифесте — `__MSG_*`, строки UI — `data-i18n`.

## Onboarding (`onboarding/`)

- Тур из 3 шагов с синтетическим примером, открывается при установке
  (`onInstalled`) и из popup. Скрипт вынесен в `onboarding.js` (CSP страниц
  расширения запрещает inline).

## Метрики (синтетический корпус)

Корпус: **93 фразы** (72 позитивных, 21 негативная — невалидные контрольные
суммы, номера заказов/треков, хеши коммитов, даты, версии). Генерируется
детерминированно: `npm run gen:corpus` (seeded PRNG; контрольные цифры
вычисляются, негативы автоматически проверяются на «тишину» через движок).

| Метрика | Значение | CI-порог |
|---|---|---|
| Precision (structured) | **0.987** | ≥ 0.98 |
| Recall (structured) | **0.961** | ≥ 0.95 |
| Recall (persons) | **0.795** | ≥ 0.65 |

Числа честные: в корпус намеренно заложены слепые зоны движка (см. ниже) и
один известный false positive — голый Luhn-валидный 16-значный «номер
заказа» распознаётся как карта (medium).

## Известные ограничения

- **ФИО — эвристика, не NER.** Не ловит: одиночные фамилии («Тимербулатов»),
  имена вне словаря («Гульнара», «Хёнсу»), беглые гласные («Павла», «Льва»),
  EN-пары без титула на пороге medium. Одиночные имена из словаря дают
  medium-шум («Марк», «Роман» как нарицательные).
- Паспорт с разбитой серией («45 03 123456»), IBAN в нижнем регистре и
  телефоны с точками-разделителями не детектируются — учтены в корпусе как
  промахи.
- Hex-строки с пометкой commit/sha/hash/md5 не считаются секретами
  (иначе каждый git-хеш был бы «утечкой»).
- Инструмент **снижения риска** утечки ПДн, а не сертифицированное средство
  защиты информации.

## Команды

```bash
npm test            # vitest: движок, vault (round-trip property), content, настройки
npm run gen:corpus  # перегенерировать tests/corpus/synthetic-corpus.json
npm run build       # dist/ = загружаемое расширение (content+background+popup+options+ассеты)
npm run build:engine # dist/engine.js — движок отдельным ES-модулем
npm run gen:icons   # перегенерировать icons/*.png (zero-dep PNG на zlib)
npm run zip         # pii-firewall-<версия>.zip для маркетплейса (после build)
```

Загрузка: `chrome://extensions` → «Режим разработчика» → «Загрузить
распакованное» → папка `dist/`.

## Комплаенс

- Все тестовые/демо-данные **синтетические**: сгенерированы, контрольные
  цифры вычислены; реальные реквизиты не используются.
- Runtime-зависимостей нет; dev-зависимости — MIT (vite, vitest, typescript, jsdom).
- `manifest.json` запрашивает только `storage` (локальное) — ни одного
  сетевого разрешения, никаких `host_permissions`.
