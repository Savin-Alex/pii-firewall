# Ручной E2E-чек-лист (M3)

Все данные в примерах — **синтетические** (контрольные суммы вычислены).

## Подготовка

1. `npm run build`
2. Chrome → `chrome://extensions` → «Режим разработчика» → «Загрузить распакованное» → папка `dist/`
3. Проверить: у расширения нет ошибок; в `chrome://extensions/shortcuts` видны Alt+Shift+M / Alt+Shift+U.

## Тестовый текст

```
Привет! Я Иванов Иван Иванович, мой ИНН 7712345671, СНИЛС 123-456-789 64.
Пиши мне на ivan.test42@example.com или звони 8 916 123 45 67.

Второй абзац — проверка многострочности.
```

## Сценарий на каждом сайте

Обязательные: ChatGPT, Claude. Желательные: Gemini, Perplexity, Poe.
Новые (селекторы best-effort, проверить в первую очередь): DeepSeek, Grok,
YandexGPT (Alice), GigaChat.

| # | Шаг | Ожидание |
|---|---|---|
| 1 | Открыть сайт, дождаться поля ввода | Виджет «PII Firewall» справа внизу, статус «Готов к защите» |
| 2 | Вставить тестовый текст, нажать «Замаскировать» | В поле: `[PERSON_1]`, `[RU_INN_1]`, `[RU_SNILS_1]`, `[EMAIL_1]`, `[PHONE_1]` + инструкция про метки; чипы по типам; оба абзаца на месте |
| 3 | Нажать «Замаскировать» повторно | «ПДн не обнаружены», инструкция **не** задублировалась |
| 4 | «Восстановить» | Исходный текст восстановлен побайтово, инструкция удалена |
| 5 | Замаскировать снова и отправить сообщение | Ответ модели содержит метки `[PERSON_1]`… |
| 6 | Выделить фрагмент ответа с метками → «Восстановить» | Тост «Восстановлено в буфер обмена»; в буфере — оригинальные значения |
| 7 | Новый чат: замаскировать → отправить (URL меняется на /c/…) → выделить ответ → «Восстановить» | Работает после смены URL (миграция draft→chat) |
| 8 | Второй таб с тем же сайтом, «Восстановить» текст из первого таба | Отказ: «Неизвестных меток: N» (изоляция по табам) |
| 9 | Alt+Shift+M / Alt+Shift+U | Работают как кнопки виджета |
| 10 | Закрыть браузер целиком, открыть заново | Vault пуст (session storage очищен) — восстановление чужих меток невозможно |
| 11 | Ввести ПДн и нажать Enter **без** маскировки | Guard-модал появляется ДО отправки; «Замаскировать и отправить» маскирует и шлёт |
| 12 | Popup → счётчик «предотвращено утечек» | Растёт после каждого «Замаскировать и отправить» в guard |
| 13 | Popup → «Демо-режим» | Открывается ChatGPT, пример в буфере; вставить и пройти цикл |
| 14 | Настройки → выключить тип (напр. EMAIL) → сохранить → вернуться в чат | EMAIL больше не маскируется без перезагрузки страницы |
| 15 | Настройки → выключить guard для сайта (popup-тумблер) | Enter с ПДн больше не показывает модал на этом сайте |
| 16 | Выделить текст → ПКМ → «Замаскировать выделение» | ПДн → по-сущностно; **произвольный текст без ПДн → весь блок `[MASKED_1]`** (гибрид), в т.ч. в textarea |

## Известные ограничения (см. README)

- Селекторы сайтов могут протухать при редизайнах — правятся в `src/content/sites.ts`;
  для DeepSeek/Grok/YandexGPT/GigaChat селекторы best-effort, проверить вживую.
- Демо-режим: content-скрипт **сам вставляет** пример в поле при загрузке; если
  редактор не появился (логин-стена/очень медленная загрузка) — откат на буфер
  обмена (вставить вручную).
