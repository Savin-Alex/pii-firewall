# Ручная проверка M3 — пошаговое руководство

Все данные ниже **синтетические** (контрольные суммы вычислены). Реальные ПДн
в тестах не используем (§8.2.6).

## Тестовый текст (скопируйте целиком)

```
Привет! Я Иванов Иван Иванович, мой ИНН 7712345671, СНИЛС 123-456-789 64.
Пиши на ivan.test42@example.com или звони 8 916 123 45 67.

Проверка второго абзаца.
```

Ожидаемые сущности: PERSON, RU_INN, RU_SNILS, EMAIL, PHONE — всего 5.

## Шаг 0. Сборка

```bash
npm run build
```

Ожидание: в `dist/` три файла — `content.js`, `background.js`, `manifest.json`.

## Шаг 1. Загрузка в Chrome

1. Откройте `chrome://extensions`.
2. Включите «Режим разработчика» (тумблер справа сверху).
3. «Загрузить распакованное расширение» → выберите папку `dist/`.
4. Ожидание: карточка **PII Firewall 0.3.0** без красной кнопки «Ошибки».
5. Нажмите ссылку **service worker** на карточке → откроется консоль SW.
   Ожидание: пусто, без красных ошибок.
6. Откройте `chrome://extensions/shortcuts`.
   Ожидание: «Замаскировать ПДн…» = Alt+Shift+M, «Восстановить…» = Alt+Shift+U
   (на macOS это ⌥⇧M / ⌥⇧U). Если пусто — назначьте вручную здесь же.

## Шаг 2. Базовый цикл на ChatGPT

1. Откройте chatgpt.com (залогиньтесь).
2. Ожидание: справа внизу виджет «PII Firewall», статус «Готов к защите».
   - Если виджета нет: F12 → Console → ищите строку
     `PII Firewall initialized for chatgpt`. Нет строки — см. «Диагностика».
3. Вставьте тестовый текст в поле ввода.
4. Нажмите **«Замаскировать»**.
   Ожидание:
   - текст в поле: `Привет! Я [PERSON_1], мой ИНН [RU_INN_1], СНИЛС [RU_SNILS_1]…`,
     `[EMAIL_1]`, `[PHONE_1]`;
   - оба абзаца на месте, **пустые строки не размножаются** (между строками
     не появляются лишние отступы — ни после первого маскирования, ни после
     цикла маскирование→восстановление→маскирование);
   - в конце добавлена одна инструкция «Сохраняй метки…»;
   - в виджете чипы: PERSON, RU_INN, RU_SNILS, EMAIL, PHONE;
   - статус: «Защищено сущностей: 5».
5. Нажмите «Замаскировать» **ещё раз**.
   Ожидание: тост «ПДн не обнаружены», инструкция НЕ задублировалась.
6. Нажмите **«Восстановить»** (ничего не выделяя).
   Ожидание: в поле исходный текст **побайтово**, инструкция удалена.

## Шаг 3. Отправка и восстановление ответа

1. Снова «Замаскировать» и отправьте сообщение (обычной кнопкой сайта).
2. Ожидание: ответ модели содержит метки `[PERSON_1]`, `[RU_INN_1]`… как текст.
3. Выделите мышью фрагмент ответа с метками.
4. Нажмите «Восстановить» в виджете.
   Ожидание: тост «Восстановлено в буфер обмена».
5. Вставьте (Cmd+V) в любое поле/заметки.
   Ожидание: метки заменены на исходные значения.

## Шаг 4. Смена URL нового чата (миграция draft → chat)

1. Начните **новый чат** (URL станет `chatgpt.com/`).
2. Вставьте текст → «Замаскировать» → отправьте.
3. Ожидание: URL сменился на `chatgpt.com/c/<id>`.
4. Выделите метки в ответе → «Восстановить» → вставьте из буфера.
   Ожидание: значения восстановлены — mapping переехал вместе с чатом.

## Шаг 5. Изоляция табов и чатов

1. Скопируйте **замаскированный** текст из первого таба.
2. Откройте второй таб с новым чатом, вставьте его в поле ввода.
3. Нажмите «Восстановить».
   Ожидание: текст НЕ восстановился, тост «Неизвестных меток: N» —
   чужая сессия не читается. Это правильное поведение.

## Шаг 6. Горячие клавиши

1. Вставьте тестовый текст, нажмите ⌥⇧M (Alt+Shift+M).
   Ожидание: маскирование как по кнопке.
2. ⌥⇧U — восстановление.
   Не сработало — проверьте назначение в `chrome://extensions/shortcuts`
   и что фокус в окне Chrome.

## Шаг 7. Автоочистка vault

1. Замаскируйте и отправьте что-нибудь.
2. Полностью выйдите из Chrome (Cmd+Q), откройте заново, вернитесь в чат.
3. Выделите метки в старом ответе → «Восстановить».
   Ожидание: «Неизвестных меток: N» — session storage очищен вместе с
   браузером. Это заявленное поведение (реверсивность без данных на диске).

## Шаг 8. Остальные сайты

Повторите шаги 2–4 на **claude.ai** (обязательный минимум по спеке), затем
по возможности gemini.google.com, perplexity.ai, poe.com. Ломаться будут
прежде всего селекторы — чинится правкой `src/content/sites.ts`.

## Шаг 9. Приватность хранилища (заодно материал для демо)

В консоли **service worker** выполните:

```js
chrome.storage.session.get(null).then(console.log)
```

Ожидание: только ключи `vault_<tab>_<host><path>` с парами
`{original, placeholder, type}` — сырого текста промптов нигде нет.

## Диагностика

| Симптом | Что смотреть |
|---|---|
| Виджет не появился | F12 → Console: есть ли `PII Firewall initialized`? Нет — DevTools → Sources → Content scripts: инжектирован ли `content.js`. Проверьте, что URL сайта попадает в `matches` манифеста |
| «Хранилище недоступно — отмена» | Консоль SW: ошибки `setAccessLevel`? Карточка расширения: SW «активен»? Перезагрузите расширение (⟳) и страницу |
| Маскирование «в никуда» (текст не изменился) | Виджет нашёл не тот редактор — поправьте селектор `editor` в `src/content/sites.ts` |
| Кнопка сайта «отправить» не видит текст | Редкий случай для React-textarea: проверьте, что после записи поле реагирует на ввод; сообщите разметку — подкрутим `writeEditor` |
| После правки кода ничего не поменялось | Тройной ритуал: `npm run build` → ⟳ на карточке расширения → перезагрузка страницы сайта. Пропуск любого шага = старый код |
| Хоткеи молчат | `chrome://extensions/shortcuts` (могли не назначиться из-за конфликта), затем консоль SW на ошибки `sendMessage` |
